# L10: Cross-Project Knowledge Sharing

**Priority**: Medium
**Status**: done

## Overview
Patterns for sharing knowledge between different projects via Qdrant.

## Problem
- Research done in one project may be useful in another
- No clear pattern for cross-project retrieval
- Collection naming and organization unclear

## Tasks
- [x] Define collection naming convention
- [x] Document cross-project retrieval pattern
- [x] Identify shared vs project-specific collections

## Results

### Collection Strategy

**Shared Collections:**
| Collection | Content | Used By |
|------------|---------|---------|
| `lineage_research` | General knowledge | All projects |
| `code_test` | Code snippets | Development |

**Project-Specific Collections:**
| Collection | Content | Project |
|------------|---------|---------|
| `midge_research` | Trading signals | MIDGE |
| `gemini_agentic_cli` | CLI-specific | This project |

### Cross-Project Retrieval Pattern

**Step 1: Check shared collection first**
```bash
python ~/.claude/scripts/qdrant-peek.py peek -c lineage_research -q "your topic" -l 3
```

**Step 2: Check project-specific if needed**
```bash
python ~/.claude/scripts/qdrant-peek.py peek -c [project_collection] -q "your topic" -l 3
```

**Step 3: Combine results**
Use lineage-retrieve skill with multiple collections if needed.

### Naming Convention

`[project-name]_[type]`
- `lineage_research` - Shared general research
- `midge_research` - MIDGE-specific research
- `wardenclyffe_research` - Wardenclyffe-specific research
- `code_index` - Code snippets (shared)

### Already Implemented

The lineage-retrieve skill supports `--collection` parameter:
```bash
python ~/.claude/scripts/qdrant-semantic-search.py --collection [any_collection] --query "topic"
```

Skills default to `lineage_research` but can target any collection.

## Handoff
Complete. Cross-project sharing is already supported via collection parameter. Naming convention documented.
